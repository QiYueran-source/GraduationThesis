前言：基于大模型部署框架Ollama，我们可以根据自身需求创建并部署自定义的本地大模型。Ollama的 Modelfile 是一种配置文件，用于定义和管理Ollama平台上的模型。借助Modelfile，我们可以创建新模型或调整现有模型，以应对一些特殊的应用场景。本文将介绍如何通过Ollama的Modelfile文件自定义大模型，并以金融助手为例，展示如何创建一个自定义大模型。

本文为Ollama系列的第四篇文章。

Modefile

Ollama 的 Modelfile 是一种配置文件，用于定义和管理 Ollama 平台上的大模型。它类似于 Docker 中的 Dockerfile，通过一系列指令来指定模型的构建过程，从而实现模型的定制化和本地部署。

Modelfile 的格式如下：

指令描述
FROM（必需）定义要使用的基础模型。
PARAMETER设置 Ollama 运行模型的参数。
TEMPLATE要发送给模型的完整提示模板。
SYSTEM指定将在模板中设置的系统消息。
ADAPTER定义要应用于模型的（Q）LoRA 适配器。
LICENSE指定法律许可。
MESSAGE指定消息历史。

1）示例

一个创建马里奥助手的 Modelfile 示例（官方示例）：

FROM llama3.2
# sets the temperature to 1 [higher is more creative, lower is more coherent]
PARAMETER temperature 1
# sets the context window size to 4096, this controls how many tokens the LLM can use as context to generate the next token
PARAMETER num_ctx 4096

# sets a custom system message to specify the behavior of the chat assistant
SYSTEM You are Mario from super mario bros, acting as an assistant.

基于上述的 Modelfile，按照以下步骤创建本地大模型：

  1. 将上述的 Modelfile 示例保存为一个文件（例如 Modelfile）。
  2. 使用命令 ollama create choose-a-model-name -f <文件位置，例如 ./Modelfile>。
  3. 使用命令 ollama run choose-a-model-name。
  4. 开始使用模型

要查看给定模型的 Modelfile，可使用 ollama show --modelfile 命令查看：

> ollama show --modelfile llama3.2
# Modelfile generated by "ollama show"
# To build a new Modelfile based on this one, replace the FROM line with:
# FROM llama3.2:latest
FROM /Users/pdevine/.ollama/models/blobs/sha256-00e1317cbf74d901080d7100f57580ba8dd8de57203072dc6f668324ba545f29
TEMPLATE """{{ if .System }}<|start_header_id|>system<|end_header_id|>

{{ .System }}<|eot_id|>{{ end }}{{ if .Prompt }}<|start_header_id|>user<|end_header_id|>

{{ .Prompt }}<|eot_id|>{{ end }}<|start_header_id|>assistant<|end_header_id|>

{{ .Response }}<|eot_id|>"""
PARAMETER stop "<|start_header_id|>"
PARAMETER stop "<|end_header_id|>"
PARAMETER stop "<|eot_id|>"
PARAMETER stop "<|reserved_special_token"

2）FROM

FROM 命令在 Ollama 的 Modelfile 中是必需的，它定义了在创建模型时要使用的基础模型。通过 FROM 命令，用户可以指定一个预训练模型作为起点，从而在此基础上进行进一步的定制和优化

FROM 指令定义了在创建模型时要使用的基础模型：

FROM <模型名称>:<标签>

基模型可以通过以下三种方式获取：

1. 从现有模型构建

用户可以从 Ollama 提供的模型库中选择一个现有的模型作为基础模型。这些模型通常已经经过预训练，可以直接用于进一步的定制。Ollama 的基础模型列表，可从 Ollama 的官网查询（https://ollama.com/library）。

详细的 Ollama 基础模型下载步骤，可参考前文：大模型本地部署开源框架 Ollama 介绍

从现有模型构建 FROM 命令示例：FROM llama3.2

2. 从 Safetensors 模型构建

Safetensors 是一种用于存储和加载深度学习模型权重的文件格式，在安全性和加载效率方面具有显著优势，特别适用于需要处理大型模型文件并关注安全性的场景。通过 Hugging Face 等平台，可以轻松找到并下载所需基模型的 Safetensors 文件。

从 Safetensors 模型构建的 FROM 命令格式：

FROM <模型目录>

模型目录应包含基模型的 Safetensors 权重文件。目前支持从 Safetensors 文件构建的基模型包括：

  • Llama（包括 Llama 2、Llama 3、Llama 3.1 和 Llama 3.2）
  • Mistral（包括 Mistral 1、Mistral 2 和 Mixtral）
  • Gemma（包括 Gemma 1 和 Gemma 2）
  • Phi3

3. 从 GGUF 文件构建

GGUF（GPT-Generated Unified Format）是一种专为大型语言模型设计的二进制文件格式，由开源项目 llama.cpp 的创始人 Georgi Gerganov 提出。它旨在解决大模型在存储、加载、兼容性和扩展性方面的挑战。基模型的 GGUF 文件，可以访问 Hugging Face 进行下载。

从 GGUF 文件构建构建的 FROM 命令格式：

FROM ./ollama-model.gguf

GGUF 文件位置应指定为绝对路径或相对于 Modelfile 位置的相对路径。

3）PARAMETER

PARAMETER 指令用于设置模型运行时的各种参数。这些参数可以影响模型的生成文本的质量、多样性和其他行为。

PARAMETER 指令格式：PARAMETER <参数> <参数值>

参数及其值如下：

  1. mirostat：
    启用 Mirostat 采样以控制困惑度。（默认值：0，0 = 禁用，1 = Mirostat，2 = Mirostat 2.0）
  2. mirostat_eta
    影响算法对生成文本的反馈响应速度。较低的学习率会导致调整更慢，而较高的学习率会使算法更灵敏。（默认值：0.1）
  3. mirostat_tau
    控制输出的连贯性和多样性平衡。较低的值将使文本更集中和连贯。（默认值：5.0）
  4. num_ctx
    设置用于生成下一个标记的上下文窗口大小。（默认值：2048）
  5. repeat_last_n
    设置模型回溯多远以防止重复。（默认值：64，0 = 禁用，-1 = num_ctx）
  6. repeat_penalty
    设置对重复的惩罚强度。较高的值（例如 1.5）将更强烈地惩罚重复，而较低的值（例如 0.9）将更宽容。（默认值：1.1）
  7. temperature
    模型的温度。提高温度会使模型回答更具创造性。（默认值：0.8）
  8. seed
    设置用于生成的随机数种子。设置为特定数字会使模型对相同提示生成相同文本。（默认值：0）
  9. stop
    设置停止序列。遇到此模式时，LLM 将停止生成文本并返回。可以通过在模型文件中指定多个单独的 stop 参数来设置多个停止模式。
  10. tfs_z
    尾部自由采样用于减少输出中较不可能标记的影响。较高的值（例如 2.0）将更多地减少影响，而值为 1.0 则禁用此设置。（默认值：1）
  11. num_predict
    生成文本时预测的最大标记数。（默认值：-1，无限生成）
  12. top_k
    减少生成无意义内容的概率。较高的值（例如 100）将给出更多样化的答案，而较低的值（例如 10）将更保守。（默认值：40）
  13. top_p
    与 top-k 配合使用。较高的值（例如 0.95）将导致文本更多样化，而较低的值（例如 0.5）将生成更集中和保守的文本。（默认值：0.9）
  14. min_p
    作为 top_p 的替代，旨在确保质量和多样性的平衡。参数 p 表示相对于最可能标记的概率，一个标记被考虑的最小概率。例如，当 p=0.05 且最可能标记的概率为 0.9 时，概率小于 0.045 的 logits 将被过滤掉。（默认值：0.0）

4）TEMPLATE

TEMPLATE 传递给模型的完整提示模板。它可以包括（可选地）系统消息、用户消息以及模型的响应。模板使用 Go 模板语法。Ollama 的模板引擎，详情可阅读本系列的前文：大模型部署框架 Ollama：模板引擎

模板变量：

1. {{ .System }}：用于指定自定义行为的系统消息。

2. {{ .Prompt }}：用户提示消息。

3. {{ .Response }}：模型的响应。在生成响应时，此变量之后的文本将被忽略。

模板示例：

TEMPLATE """{{ if .System }}<|im_start|>system
{{ .System }}<|im_end|>
{{ end }}{{ if .Prompt }}<|im_start|>user
{{ .Prompt }}<|im_end|>
{{ end }}<|im_start|>assistant
"""

5）SYSTEM

SYSTEM 指令用于指定模板中使用的系统消息（即指定 {{ .System }} 变量的值）：

SYSTEM """<系统消息>"""

6）ADAPTER

适配器（Adapter）是一种高效的参数微调方法，特别适用于大型语言模型（LLM）。适配器通过在预训练模型的每一层或指定层中插入小型的神经网络模块，来调整模型以适应下游具体的应用任务。

ADAPTER 指令用于指定应用于基础模型的微调 LoRA 适配器。适配器的值应为绝对路径或相对于 Modelfile 的路径。基础模型由 FROM 指令指定。如果基础模型与适配器微调时所用的基础模型不同，则模型行为将不可预测。 因此，FROM 指定的基模型，应与适配器微调时所用的基础模型一致。

Safetensor 适配器

ADAPTER <safetensor 适配器的路径>

目前支持的 Safetensor 适配器：

  • Llama（包括 Llama 2、Llama 3 和 Llama 3.1）
  • Mistral（包括 Mistral 1、Mistral 2 和 Mixtral）
  • Gemma（包括 Gemma 1 和 Gemma 2）

GGUF 适配器

ADAPTER ./ollama-lora.gguf

7）LISENCE

LICENSE 指令允许你指定与本 Modelfile 一起使用的模型的法律许可，用于共享或分发。

ICENSE """
<license text>
"""

8）MESSAGE

MESSAGE 指令允许你为模型指定用于响应时的消息历史。使用 MESSAGE 指令在 Ollama 的 Modelfile 中构建对话历史，可以显著影响模型的响应方式，使其更加连贯和符合预期。

MESSAGE <角色> <消息>

有效的角色（关于角色，可阅读前文：大模型部署框架 Ollama：模板引擎）：

1. system：为模型提供 SYSTEM消息的另一种方式。

2. user：用户可能提出的问题示例。

3. assistant：模型应如何回答的示例。

Message 示例对话

MESSAGE user Is Toronto in Canada?
MESSAGE assistant yes
MESSAGE user Is Sacramento in Canada?
MESSAGE assistant no
MESSAGE user Is Ontario in Canada?
MESSAGE assistant yes

>>> 介绍一下夏普比率
当然可以。夏普比率（Sharpe Ratio）是衡量投资组合每单位风险所带来的超额回报的指标。它是以哈里·马科维茨的投资组合理论
为基础的一个重要工具。夏普比率越高，意味着在相同的风险水平下，投资组合能获得更高的超额回报；或者在相同的预期收益率下
，所承担的风险更低。

### 夏普比率的计算公式

夏普比率可以通过以下公式来计算：

\[ \text{夏普比率} = \frac{R_p - R_f}{\sigma_p} \]

其中：
- \( R_p \) 是投资组合的实际平均收益。
- \( R_f \) 是无风险收益率，通常可以使用同期的国债利率或其他被视为无风险的投资收益作为参考。
- \( \sigma_p \) 是投资组合的总波动性或标准差。

### 示例解释

假设你有一个投资组合，其平均年化回报率为12%，无风险收益率为4%，而该投资组合的标准差（风险度量）是10%。那么：

\[ \text{夏普比率} = \frac{(12\% - 4\%)}{10\%} = \frac{8\%}{10\%} = 0.8 \]

这意味着该投资组合每承担一单位风险，可以带来0.8个单位的超额回报。

### 夏普比率的应用

夏普比率广泛应用于比较不同资产或投资组合的风险调整后收益。它可以帮助投资者在多个选择中做出更为明智的决策，比如判断一
个管理者的业绩是否因管理水平而优于市场平均水平，而不是单纯依赖于高收益率。

需要注意的是，虽然夏普比率是一个很有用的工具，但它也有其局限性，例如它假定投资回报率呈正态分布，实际情况下可能并不完
全符合。此外，它也仅考虑了总波动性，并未区分不同来源的风险类型（如市场风险和信用风险）。
